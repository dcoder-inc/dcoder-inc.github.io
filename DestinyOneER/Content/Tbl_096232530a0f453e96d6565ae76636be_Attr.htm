expiration_type 
Base Entity  profile_hold 
ColumnName  expiration_type 
Definition  
How the profile hold will expire.  Possible values are: manual, onDate.  When set to ""onDate"" the system will rely on a scheduled daemon/task and the value of the ""expiration_date"" field.
 
expiration_date 
Base Entity  profile_hold 
ColumnName  expiration_date 
Definition  
The date that the profile hold expires/expired.  For manual expirations, this will normally be blank.  This field value is only used for the daemon/task that can expire a profile hold based on a scheduled date (expiration_date).
