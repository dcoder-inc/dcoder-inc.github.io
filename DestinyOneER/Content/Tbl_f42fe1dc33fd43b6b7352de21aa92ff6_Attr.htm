original_fee_lw_id 
Base Entity  fee_lw 
ColumnName  original_fee_lw_id 
Definition  
The ID of the fee lw that represents the initial fee that was first persisted to the database, which is important in understanding  a fee lw adjustment chain.  More specifically, either this column is equal to the fee_lw_id column, in which case b_adjustment must be '0', or it is equal to the one that is stored in the 'adjust to' fee lw, in which case b_adjustment must be '1' and adjust_to_fee_lw_id must be defined.
 
b_adjusted 
Base Entity  fee_lw 
ColumnName  b_adjusted 
Definition  
A flag that indicates whether this fee lw has been adjusted or not.  0: not adusted, 1: adjusted.   In other words, this column has value '1' if and only if there exists another fee lw row R defined such that the 'adjust_to_fee_lw_id'  column of row R is equal to the fee_lw_id column of the current row.
